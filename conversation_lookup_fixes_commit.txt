Fix conversation lookup issues - missing return statement and auth token handling

## Summary of Changes

Fixed critical conversation lookup issues that were causing "0 conversations found" errors and preventing proper conversation persistence for new visitors.

### Backend Fixes
- **BaseController**: Fixed missing return statement in `conversations` method that was causing it to return `nil` instead of ActiveRecord relation
- **WebsiteTokenHelper**: Enhanced `auth_token_params` method to gracefully handle missing auth tokens for new visitors
- **BaseController**: Added fallback logic to use web widget's inbox_id when auth token is empty
- **ConversationsController**: Improved `update_last_seen` endpoint with proper error handling and logging

### Frontend Fixes
- **Axios Helper**: Fixed conversation ID logging to avoid confusion between contact IDs and conversation IDs

### Root Cause
The "0 conversations found" issue was caused by:
1. Missing return statement in `conversations` method causing it to return `nil`
2. Empty auth_token_params for new visitors causing `inbox_id` to be `nil` in queries
3. Inadequate error handling in `update_last_seen` endpoint

- `app/controllers/api/v1/widget/base_controller.rb`
- `app/controllers/concerns/website_token_helper.rb`
- `app/controllers/api/v1/widget/conversations_controller.rb`
- `app/javascript/widget/helpers/axios.js`
- `conversation_persistence_checklist.md`

### Impact
- Fixes conversation persistence for new visitors without auth tokens
- Ensures robust conversation lookup with proper fallback logic
- Provides better error handling and debugging capabilities
- Maintains backward compatibility with existing functionality

These fixes ensure the conversation persistence feature works reliably for all user scenarios, including new visitors and edge cases.
