## 📊 Technical Architecture

### Conversation Persistence Flow
1. **Visitor ID Generation**: Unique ID stored in sessionStorage
2. **Redis Mapping**: Visitor mapped to conversation tokens (30-day TTL)
3. **Conversation Lookup**: Redis validation → Database fallback
4. **Message Routing**: Messages to existing conversations only
5. **Auto Cleanup**: Stale mappings cleaned on conversation resolution

### Key Components
- **VisitorConversationMapping**: Redis-based visitor tracking with validation
- **Widget::TokenService**: Conversation tokens with ID for precise validation
- **Enhanced Controllers**: Multi-step conversation lookup with fallbacks
- **Frontend State**: Vuex store maintains conversation across navigation
- **Webhook System**: Preserved and enhanced for n8n integration
