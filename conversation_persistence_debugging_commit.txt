Add Comprehensive Debugging for Conversation Persistence Issue

## Problem Identified
User reported that new conversations are being created during page navigation instead of reusing existing conversations:
- Page 1: Conversation 483 created with user messages and n8n responses ✅
- Page 2: Conversation 484 created with only 2 messages (widget open + n8n response) ❌
- Page 3: Conversation 485 created, all previous messages lost ❌

## Root Cause Analysis
The conversation persistence feature is partially working (webhook prevention is working), but the conversation lookup logic is failing to find existing conversations during page navigation.
