COMMIT NAME:
Fix conversation persistence issues - prevent multiple conversations bug

COMMIT SUMMARY:
Fixed critical conversation persistence issues where new conversations were being created on page navigation and message sending instead of maintaining existing conversations. **IMPORTANT: All existing webhook functionality for n8n integration is preserved and enhanced.**

## Key Changes:

### Backend Fixes:
- **Messages Controller**: Modified set_conversation method to return error instead of creating new conversations when none exists
- **Enhanced Logging**: Added comprehensive logging to conversation lookup, contact creation, and Redis mapping processes
- **Error Handling**: Proper error responses with NO_CONVERSATION code when conversation not found

### Frontend Improvements:
- **Message Handling**: Updated sendMessageWithData to handle NO_CONVERSATION error and create conversation first if needed
- **Page Navigation**: Added ensureConversationPersistence method to maintain conversation state after navigation
- **Comprehensive Logging**: Added detailed logging for visitor ID, conversation creation, message sending, and API requests
- **Debug Test Suite**: Created test suite to verify visitor ID persistence and conversation flow

### ✅ WEBHOOK FUNCTIONALITY PRESERVED:
**n8n Integration Status: FULLY WORKING**
- `conversation.created` webhook still fires when new conversations start
- `conversation.resolved` webhook still fires when conversations end
- All webhook payload data preserved (conversation details, page info, custom attributes)
- **IMPROVED**: Eliminates duplicate webhook spam during page navigation
- **IMPROVED**: Clean conversation lifecycle - one creation webhook, one resolution webhook per conversation

### Conversation Lifecycle with Webhooks:
1. User opens chat → New conversation → **Webhook to n8n** ✅
2. User navigates pages → Same conversation maintained → **No duplicate webhooks** ✅
3. User sends messages → Added to existing conversation → **No unwanted webhooks** ✅
4. User ends conversation → Conversation resolved → **Webhook to n8n** ✅
5. User opens chat again → New conversation → **Webhook to n8n** ✅
