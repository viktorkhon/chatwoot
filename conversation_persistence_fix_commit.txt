COMMIT NAME:
Fix conversation persistence issues - prevent multiple conversations bug

COMMIT SUMMARY:
Fixed critical conversation persistence issues where new conversations were being created on page navigation and message sending instead of maintaining existing conversations.

## Key Changes:

### Backend Fixes:
- **Messages Controller**: Modified set_conversation method to return error instead of creating new conversations when none exists
- **Enhanced Logging**: Added comprehensive logging to conversation lookup, contact creation, and Redis mapping processes
- **Error Handling**: Proper error responses with NO_CONVERSATION code when conversation not found

### Frontend Improvements:
- **Message Handling**: Updated sendMessageWithData to handle NO_CONVERSATION error and create conversation first if needed
- **Page Navigation**: Added ensureConversationPersistence method to maintain conversation state after navigation
- **Comprehensive Logging**: Added detailed logging for visitor ID, conversation creation, message sending, and API requests
- **Debug Test Suite**: Created test suite to verify visitor ID persistence and conversation flow
