Debug and Fix Conversation Creation During Page Navigation

## Problem
User reported that new conversations were being created during page navigation even without opening the widget, despite Redis persistence working correctly. This suggested that either frontend API calls or external services (like n8n automation) were triggering conversation creation.

## Root Cause Investigation
- **Frontend Analysis**: Found that `ensureConversationPersistence()` was calling `fetchOldConversations()` on every page navigation, triggering unnecessary API calls
- **Backend Analysis**: Confirmed that conversation creation only happens via explicit API calls to `/api/v1/widget/conversations`
- **Suspected External Cause**: n8n automation likely receiving `webwidget_triggered` webhooks and making API calls back to Chatwoot to create conversations

## Fixes Implemented

### 1. Removed Unnecessary API Calls During Navigation
- **File**: `app/javascript/widget/App.vue`
- **Change**: Modified `ensureConversationPersistence()` to only check existing frontend state instead of calling `fetchOldConversations()`
- **Impact**: Eliminates unnecessary server requests during page navigation, prevents potential side effects

### 2. Enhanced Debug Logging for Conversation Creation
- **File**: `app/controllers/api/v1/widget/conversations_controller.rb`
- **Change**: Added comprehensive request tracking including User-Agent, Referer, IP, and visitor ID
- **Impact**: Identifies the source of conversation creation requests (widget frontend vs external services like n8n)

### 3. Enhanced Debug Logging for Widget Events
- **File**: `app/controllers/api/v1/widget/events_controller.rb`
- **Change**: Added event tracking with request details for webwidget.triggered events
- **Impact**: Tracks when and from where widget events are created

### 4. Enhanced Frontend Debug Logging
- **File**: `app/javascript/widget/App.vue`
- **Change**: Added logging for widget events and page navigation with conversation state
- **Impact**: Tracks frontend behavior during navigation and event triggering

### 5. Cleaned Up Excessive Debug Logging
- **File**: `app/controllers/api/v1/widget/base_controller.rb`
- **Change**: Removed excessive Redis and database lookup logs while keeping essential conversation tracking
- **Impact**: Reduces log noise while maintaining debugging capabilities

## Expected Results
- **Performance**: Reduced API calls during page navigation
- **Debugging**: Enhanced ability to identify the source of conversation creation requests
- **Root Cause Detection**: Logs will show if n8n or other external services are creating conversations
