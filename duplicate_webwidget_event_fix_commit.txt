Fix Duplicate webwidget.triggered Events - Coordinate Prevention Logic

## Problem
User reported duplicate webwidget.triggered events causing duplicate conversations despite webhook prevention implementation. Frontend logs showed:
```
[Chatwoot] Sending webwidget.triggered event for NEW chat session
[Chatwoot] Widget event triggered: {eventName: 'webwidget.triggered', isWidgetTriggerEvent: true, currentRoute: 'home', isWidgetOpen: true}
[Chatwoot] Dispatching webwidget.triggered event to backend
```

## Root Cause
**Dual Processing Issue**: Both IFrameHelper and App.vue were processing webwidget.triggered events independently:

1. **IFrameHelper.onBubbleToggle** → Checks prevention logic → Sends `pushEvent('webwidget.triggered')`
2. **App.vue.createWidgetEvents** → Receives push-event → Dispatches to backend WITHOUT checking prevention

**Race Condition**: IFrameHelper marked session BEFORE sending event, but App.vue didn't check this flag until AFTER receiving the event, creating a timing issue.

## Solution Implemented

### 1. Added Prevention Logic to App.vue
**File**: `app/javascript/widget/App.vue`
**Method**: `createWidgetEvents`

- **ADDED**: Same dual-condition prevention logic as IFrameHelper
- **CHECKS**: Both session flag and conversation existence before dispatching
- **ENHANCED**: Detailed debugging logs for webhook prevention decisions

```javascript
// Enhanced webhook prevention for webwidget.triggered events
if (isWidgetTriggerEvent) {
  const sessionKey = 'chatwoot_webwidget_triggered_session';
  const conversationKey = 'chatwoot_conversation_exists';
  
  const hasTriggeredInSession = sessionStorage.getItem(sessionKey);
  const conversationExists = sessionStorage.getItem(conversationKey);
  
  // Only dispatch if both conditions are false
  if (hasTriggeredInSession || conversationExists) {
    console.log('[Chatwoot] Skipping webwidget.triggered dispatch - prevention logic');
    return;
  }
  
  console.log('[Chatwoot] Dispatching webwidget.triggered event to backend - NEW chat session');
}
```

### 2. Coordinated Session Management
**File**: `app/javascript/widget/store/modules/events.js`
**Method**: `create`

- **ADDED**: Session marking AFTER successful API dispatch
- **ENSURES**: Session flag is set only when event actually reaches backend
- **PREVENTS**: Race conditions between components

```javascript
await events.create(name);

// Mark session as triggered for webwidget.triggered events to prevent duplicates
if (name === 'webwidget.triggered') {
  sessionStorage.setItem('chatwoot_webwidget_triggered_session', Date.now().toString());
  console.log('[Chatwoot] Marked session as triggered for webwidget.triggered event');
}
```

### 3. Fixed Race Condition in IFrameHelper
**File**: `app/javascript/sdk/IFrameHelper.js`
**Method**: `onBubbleToggle`

- **REMOVED**: Premature session marking before event dispatch
- **CHANGED**: Let events store handle session marking after successful API call
- **FIXED**: Race condition between IFrameHelper and App.vue

```javascript
if (!hasTriggeredInSession && !conversationExists) {
  // Send the webwidget.triggered event for new chat session
  // Session will be marked by events store after successful dispatch
  IFrameHelper.pushEvent('webwidget.triggered');
  console.log('[Chatwoot] Sending webwidget.triggered event for NEW chat session');
}
```

### 4. Enhanced Debug Logging
**Files**: Multiple components

- **ADDED**: Detailed webhook prevention decision logging
- **INCLUDES**: Session values, conversation state, and timing information
- **HELPS**: Track exact flow and identify any remaining issues
