feat: Identify and fix root cause of n8n duplicate conversation creation

## Problem Solved ✅
User reported duplicate conversations being created when n8n responds to message_update webhooks.

## Root Cause Discovered
**n8n is calling TWO different endpoints:**

1. ✅ **CORRECT**: `/api/v1/accounts/2/conversations/541/messages` (adds message to existing conversation)
2. ❌ **PROBLEMATIC**: `/api/v1/accounts/2/conversations` (creates NEW conversation every time)

The second endpoint is the conversation creation endpoint, which explains why duplicate conversations are being created.

## Solution Implemented

**File**: `app/controllers/api/v1/accounts/conversations_controller.rb`
- Added comprehensive logging to detect when n8n incorrectly calls conversation creation endpoint
- Clear error messages identifying the exact problem
