fix: Resolve widget initialization timing error with safe locale access

## Summary
Fixed critical widget initialization error where `buildSearchParamsWithLocale` function was accessing `window.WOOT_WIDGET.$root.$i18n.locale` before the widget was fully initialized, causing "Cannot read properties of undefined (reading '$root')" error.

## Key Issues Resolved
- Fixed TypeError when accessing `$root` during early widget initialization
- Added safe fallback chain for locale detection with multiple sources
- Enhanced error handling to provide informative warnings instead of breaking errors
- Improved widget initialization reliability across different loading scenarios

## Technical Changes

### Frontend (app/javascript/widget/helpers/urlParamsHelper.js)
- **Enhanced buildSearchParamsWithLocale function**: Added comprehensive fallback chain for locale detection
- **Safe access pattern**: Protected `window.WOOT_WIDGET.$root` access with existence checks
- **Multiple fallback sources**: 
  1. `window.WOOT_WIDGET.$root.$i18n.locale` (when fully initialized)
  2. `window.WOOT_WIDGET.locale` (alternative widget property)  
  3. `navigator.language.split('-')[0]` (browser language)
  4. `'en'` (final fallback)
- **Try-catch protection**: Graceful error handling with warning messages

### Frontend (app/javascript/widget/store/modules/conversationAttributes.js)
- **Enhanced error handling**: Specific detection for `$root` access timing issues
- **Informative warnings**: User-friendly messages for expected initialization timing issues
- **Clean state management**: Proper state cleanup when errors occur

## Root Cause Analysis
**Widget Initialization Timing Issue**: The Vue app's mounted hook executes immediately when the iframe loads, but `window.WOOT_WIDGET.$root` isn't available until `window.onload` completes in the main widget initialization. This created a race condition where API calls requiring locale information would fail.

**Call Stack**:
1. Widget iframe loads → Vue app mounts
2. `App.vue` mounted hook → dispatches `conversationAttributes/getAttributes`
3. `getConversationAPI()` → calls `buildSearchParamsWithLocale()`
4. Function tries to access `window.WOOT_WIDGET.$root.$i18n.locale`
5. **ERROR**: `$root` undefined because main widget initialization incomplete

- ✅ **Graceful degradation**: Widget functions even during early initialization phases
