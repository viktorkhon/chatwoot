## Fix Widget Initialization: Replace require() with ES6 Import

### Summary
Fixed critical "ReferenceError: require is not defined" error preventing widget initialization by replacing Node.js-style require statement with proper ES6 import syntax.

### Problem Resolved
- **Error**: `ReferenceError: require is not defined at App.vue:359:37`
- **Location**: `initializeVisitorTracking()` method in widget App.vue
- **Impact**: Complete widget initialization failure

### Root Cause
The `initializeVisitorTracking` method was using Node.js-style require syntax:
```javascript
const { generateVisitorId } = require('widget/helpers/utils');
```
This syntax is not supported in browser environments and caused runtime errors.

### Solution Applied

**Fixed Import Statement**:
- Added `generateVisitorId` to existing ES6 import from utils
- Changed: `import { IFrameHelper, RNHelper } from 'widget/helpers/utils';`
- To: `import { IFrameHelper, RNHelper, generateVisitorId } from 'widget/helpers/utils';`

**Removed require() Statement**:
- Deleted: `const { generateVisitorId } = require('widget/helpers/utils');`
- Function now available through proper ES6 import

### Files Modified
- `app/javascript/widget/App.vue` - Fixed import and removed require statement
