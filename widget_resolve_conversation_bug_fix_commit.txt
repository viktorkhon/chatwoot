Fix widget reset TypeError by creating window.$chatwoot in widget context

## Problem
Widget resolve conversation and start new conversation functions were failing with:
`TypeError: Cannot read properties of undefined (reading 'reset')`

This occurred because the widget was trying to call `window.$chatwoot.reset()` directly,
but the widget runs in iframe mode where `window.$chatwoot` exists in the parent window,
not in the iframe context.

## Root Cause Analysis
The widget has two execution modes:
1. **Iframe Mode**: `window.self !== window.top` - widget runs in iframe, no direct access to parent's $chatwoot
2. **Direct Mode**: `window.self === window.top` - widget runs in same window, has direct access to $chatwoot

The broken branch was missing the widget-specific `window.$chatwoot` object that handles both modes.

## Solution
Implemented the correct architecture pattern by creating a widget-specific `window.$chatwoot`
object that handles both iframe and direct modes:

1. **Created widget-specific $chatwoot object** (`app/javascript/entrypoints/widget.js`):
   - In iframe mode: Sends reset message to parent via `IFrameHelper.sendMessage({ event: 'reset' })`
   - In direct mode: Performs local reset (clears storage, navigates to pre-chat form)

2. **Added reset event handler in SDK** (`app/javascript/sdk/IFrameHelper.js`):
   - Handles reset messages from widget iframe
   - Calls parent window's `window.$chatwoot.reset()` when receiving reset event

## Technical Details
- Widget creates its own `window.$chatwoot` object that intelligently handles both execution modes
- Uses existing `IFrameHelper.sendMessage()` mechanism for iframe-to-parent communication
- Maintains backward compatibility with direct mode via local reset implementation
- Follows established patterns used by other widget-to-parent communications

## Files Changed
- `app/javascript/entrypoints/widget.js` - Created widget-specific $chatwoot object with reset method
- `app/javascript/sdk/IFrameHelper.js` - Added reset event handler for iframe communication

## Testing
The fix handles both widget execution modes:
- **Iframe Mode**: Widget sends reset message to parent, parent calls its own $chatwoot.reset()
- **Direct Mode**: Widget performs local reset operations directly
